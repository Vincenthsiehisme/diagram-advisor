# Diagram Type Index — 視角對照與決策樹

Phase 1 必讀。確認視角後，再按需載入對應的完整定義檔。

---

## 視角快速對照

| 圖的種類 | 核心問題 | 主要受眾 | 典型觸發語 |
|---------|---------|---------|-----------|
| 系統架構圖 | 有哪些系統、邊界在哪、怎麼部署 | SA / RD | 「畫架構圖」「系統邊界」「怎麼部署」 |
| 資料流圖（DFD） | 資料從哪來、往哪去、誰擁有、誰讀誰寫 | 資料架構師、跨 Squad PM | 「資料怎麼流」「誰 own 這個 table」「讀寫關係」 |
| 作業流圖 | 人怎麼操作、步驟順序、判斷點在哪 | PM / 業務 / OP | 「畫流程圖」「操作步驟」「誰先做什麼」 |

三種圖**不合併**。合併的結果是對每個受眾都讀不懂。

---

## 視角選擇決策樹

```
使用者說「畫圖」
    ↓
受眾是 SA / RD？
    是 → 系統架構圖
    否 ↓
問題是「資料誰 own、怎麼流、誰讀誰寫」？
    是 → 資料流圖（DFD）
    否 ↓
問題是「人怎麼操作、步驟順序、判斷點」？
    是 → 作業流圖
    否 ↓
說「都要」或「通用版」？
    是 → 分開畫三張，建議順序：系統架構圖 → 資料流圖 → 作業流圖
```
